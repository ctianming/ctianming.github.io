# Ethernaut 部分题解

# GateKeeperOne（有问题）

题目：

gateOne: 要求使用攻击合约

gateTwo: 要求 gasleft 是 8191 的整数倍

gateThree: 要求获得一个_gatekey

对于本题，我之前找了很久的 gas，但是一直没法通过。

然后写了如上一个 for 循环，想要暴力通过（因为 call 不会回退），在本地测试通过，success 返回 true，但是很奇怪的是，在连接上钱包之后却返回 false。

所以这里出现了问题了。。。

# GateKeeperTwo

先看一下代码

gateOne：要求使用攻击合约

gateTwo：要求合约仅有构造函数

gateThree：要求 msg.sender 的异或运算结果符合要求

仅需在构造函数中写入

uint64 key = uint64(bytes8(keccak256(abi.encodePacked(address(this)))));
    key = key ^ 0xffffffffffffffff;

即可完成攻击

# Naught coin

从题目中我们可以发现

function transfer(address _to, uint256 _value) override public lockTokens returns(bool) {
super.transfer(_to, _value);
  }

题目为 transfer 函数添加上了 lockTokens 的修饰器，但是不必突破它

题目引用

import 'openzeppelin-contracts-08/token/ERC20/ERC20.sol';

我们找到这个仓库，发现转账函数不只 transfer, 还有一个 transferFrom，于是我们只需通过该函数将币转出即可

同时需要注意使用 transferFrom 需要提前取得授权（通过 approve 函数和 allowance 函数）

执行：

await contract.transferFrom(player, "instance", await contract.balanceOf(player));

# Alien Codex

本题在于找到 owner 在 codex 中的存储位置，然后运行 revise 函数将相应的 owner 改为自己。由于该合约继承 Ownable 合约，所以 owner 应该存储在第 0 个槽位。而 codex 的长度存储在第一个槽位。

执行

await web3.eth.getStorageAt(instance.1)

发现 codex 的长度是 0；那么就需要执行 retract 函数，使 codex 下溢，长度变为 2^256.

接下来通过 keccak256 (bytes32 (1)) 得到 codex 的实际内容开始存储的位置 x；那么，就得到 owner 存储的位置为 2^256-x。

最后，只需根据所得位置，执行 revise 即可。

# Denial

这个题目，一眼就抓住那个 receiver，只需利用好 receive 阻止 owner 提走资金即可完成。

攻击合约：

如图，该题中，owner 提钱时调用 withdraw，在提走资金前会先给我转钱。那么，只需在攻击合约中添加一个 receive，并使 owner 出现递归调用，就可以耗尽他的 gas，进而阻止提取资金
